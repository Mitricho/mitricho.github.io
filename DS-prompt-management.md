Отличная задача! Ключ в том, чтобы создать **структурированный процесс разработки** с четкими правилами, этапами и обратной связью для ИИ, компенсируя его «слепоту». Вот многоуровневая методология:

### 1. Фундамент: Система Дизайн-токенов и API первого дизайна
ИИ не должен «придумывать» дизайн с нуля каждый раз. Ему нужна система.

*   **Создайте библиотеку дизайн-токенов в формате JSON:**
    ```json
    {
      "colors": {
        "primary": "#2563eb",
        "secondary": "#7c3aed",
        "background": "#f8fafc"
      },
      "spacing": {
        "unit": "0.5rem",
        "sectionPadding": "3rem"
      },
      "typography": {
        "fontFamily": "Inter, sans-serif",
        "h1": { "size": "2.5rem", "weight": "700" }
      },
      "components": {
        "button": {
          "padding": "0.75rem 1.5rem",
          "borderRadius": "0.375rem",
          "variants": ["primary", "secondary", "outline"]
        }
      }
    }
    ```
*   **Задача для ИИ:** Генерировать код, опираясь **строго** на эти токены. Например: `background-color: var(--color-primary)`. Это обеспечивает консистентность и управляемость.

### 2. Архетипы и Шаблоны: «Конструктор» вместо чистого листа
Вместо «создай страницу» задача должна звучать как **«собери страницу из готовых блоков»**.

*   **Создайте библиотеку архетипов (Figma-like для ИИ):**
    *   `HeroSection`: обязательные поля (заголовок, текст, 2 кнопки), опциональные (фон, изображение).
    *   `FeaturesGrid`: заголовок, подзаголовок, массив фич (иконка, заголовок, текст).
    *   `TestimonialsCarousel`: массив отзывов (аватар, текст, имя).
    *   `ContactForm`: поля (имя, email, сообщение), настройки (endpoint API, текст кнопки).

*   **Задача для ИИ:** Получает на вход JSON-схему будущей страницы:
    ```json
    {
      "page": {
        "title": "Главная",
        "sections": [
          { "type": "HeroSection", "data": { "title": "Привет мир", ... } },
          { "type": "FeaturesGrid", "data": { "features": [...] } }
        ]
      }
    }
    ```
    Его задача — сопоставить `type` с известным ему шаблоном и «наполнить» его переданными `data`.

### 3. Процесс «Разделенной ответственности»: Изоляция проблем
Верстку и логику нужно разделять.

*   **Этап 1: Семантическая HTML-структура.** ИИ генерирует **только HTML** с классами и data-атрибутами. Без CSS. Это легко проверить на валидность и структуру.
    *   Запрос: «Создай семантическую разметку для FeaturesGrid с 3 колонками. Используй классы `.features-grid`, `.feature-item`».
*   **Этап 2: CSS на основе дизайн-токенов.** Передаете ИИ сгенерированный HTML и просите: «Напиши CSS для этого HTML, используя следующие дизайн-токены: ...». ИИ видит структуру классов и применяет правила.
*   **Этап 3: Интерактивность (JS).** Четкое ТЗ: «Добавь JS для карусели testimonials. Используй классы `.testimonial-item`, `.next-btn`. Функционал: плавная прокрутка по клику».

### 4. Система Верификации и Обратной Связи: «ИИ-линтер»
Поскольку ИИ не видит результат, ему нужно дать инструменты для самопроверки.

*   **Создайте набор четких правил в виде чек-листа, который ИИ проходит перед выдачей кода:**
    1.  Все ли изображения имеют атрибут `alt`?
    2.  Все ли интерактивные элементы доступны с клавиатуры (tabindex)?
    3.  Соответствует ли структура заголовков (`h1` -> `h2` -> `h3`)?
    4.  Используются ли CSS-переменные из токенов?
    5.  Есть ли комментарии в ключевых местах кода?

*   **ИИ должен включать вывод этого чек-листа в свой ответ:** «Проверка: 1) ✅, 2) ✅, 3) ⚠️ (пропущен h2), 4) ✅...». Это дисциплинирует модель.

### 5. Интеграция с Headless CMS: Управляемый контент
Структура данных должна приходить извне.

*   **Схема:** ИИ генерирует не статический текст, а **шаблоны с плейсхолдерами**.
    *   Вместо `<h1>Лучший продукт</h1>` -> `<h1>{{ page.hero.title }}</h1>`.
*   **Задача:** «Создай компонент HeroSection, где заголовок, текст и ссылки берутся из объекта `heroData`». Это делает контент полностью управляемым через CMS.

### Пример итогового рабочего процесса:

1.  **Планировщик (Человек или ИИ высокого уровня):** Генерирует JSON-схему страницы (п.2).
2.  **Архитектор (ИИ-кодер, получающий схему + дизайн-токены):**
    *   *Шаг 1:* Пишет семантический HTML по схеме.
    *   *Шаг 2:* Проходит чек-лист валидации (п.4).
    *   *Шаг 3:* Пишет CSS, используя токены (п.1).
    *   *Шаг 4:* Добавляет базовый JS для интерактивных блоков.
    *   *На выходе:* Чистый, структурированный код с комментариями.
3.  **Сборщик (Человек или система сборки):** Интегрирует результат с CMS, подставляет реальный контент, запускает финальные тесты (можно даже с помощью скриншотного тестирования, чтобы дать обратную связь ИИ).

### Технические инструменты для реализации:
*   **Prompt-инженерия:** Создайте многошаговые промпты с строгими инструкциями.
*   **Custom GPT / Агенты:** Настройте агента с файлами-контекстом (дизайн-токены.json, шаблоны.html, правила.md).
*   **Low-code платформа на основе ИИ:** Система, где вы визуально собираете схему страницы, а ИИ под капотом генерирует качественный код по описанным правилам.

**Суть:** Вы не просите ИИ «увидеть» дизайн. Вы создаете для него **детерминированную среду с правилами**, где его задача сводится к трансляции структурированных данных в структурированный код, используя заранее подготовленные кирпичики. Это превращает проблему творчества в проблему следования инструкциям, с которой современные ИИ справляются превосходно.
